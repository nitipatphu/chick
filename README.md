## 🎯 ทำโปรเจกต์นี้ไปทำไม?
1. อยากมีเว็บขายอุปกรณ์กีฬาที่ใช้งานง่ายๆ
2. ให้ลูกค้าเข้ามาค้นหาและกดสั่งซื้อของได้สะดวกสบายที่สุด
3. ฝั่งคนขาย (แอดมิน/พนักงาน) ก็ต้องมีระบบหลังบ้านที่จัดการสต็อกและออเดอร์ได้ง่ายๆ ไม่ปวดหัว

## 📌 ที่มาที่ไป
เดี๋ยวนี้ใครๆ ก็ซื้อของออนไลน์กันทั้งนั้น โดยเฉพาะพวกอุปกรณ์กีฬาที่มีของจุกจิกเยอะมาก พวกเราเลยอยากทำระบบที่ช่วยให้ลูกค้าเข้ามาดูของ เลือกไซส์ เลือกสี และกดสั่งซื้อได้เลย ส่วนหลังบ้านก็ออกแบบมาให้จัดการข้อมูลได้เป๊ะปัง ไม่มีตกหล่นครับ

---

## 👤 ใครทำอะไรได้บ้างในระบบนี้? (System Scope)

### ผู้ใช้งาน 3 ระดับ
1. **Customer (ลูกค้าทั่วไป)** 
   - สมัครสมาชิก / ล็อกอิน
   - เข้ามาดูของ ค้นหาสินค้า
   - กดหยิบใส่ตะกร้าและสั่งซื้อ
   - อัปโหลดสลิปโอนเงิน
   - ดูประวัติว่าตัวเองเคยสั่งอะไรไปแล้วบ้าง
   
2. **Staff (พนักงาน)** 
   - ล็อกอินด้วยสิทธิ์พนักงาน
   - คอยเช็คออเดอร์ที่เข้ามาใหม่
   - เข้ามาแพ็คของ, อัปเดตสถานะจัดส่ง และกรอกเลขพัสดุ (Tracking)
   - เช็คสต็อกสินค้า

3. **Administrator (ผู้ดูแลระบบ)**
   - สิทธิ์ใหญ่สุด ทำได้ทุกอย่าง
   - จัดการสินค้า (เพิ่ม/ลบ/แก้ไขข้อมูลสินค้า)
   - ดูแลเรื่องหมวดหมู่สินค้า
   - ตรวจสอบสลิปโอนเงินแล้วกดยืนยันให้ลูกค้า
   - จัดการข้อมูลผู้ใช้งานในระบบ
   - ดูรายงานสรุปยอดขาย

## 🔧 เครื่องมือที่พวกเราใช้ (Tech Stack)
อันนี้ปรับปรุงใหม่ตามระบบจริงที่เราเขียนกันเลยครับ:

- **Frontend (หน้าเว็บ):** React, Tailwind CSS (เขียน UI สวยๆ สไตล์ Modern)
- **Backend (ระบบหลังบ้าน):** Node.js รันด้วย Express
- **Database (ฐานข้อมูล):** เก็บข้อมูลลงไฟล์ `db.json` (จำลองฐานข้อมูลให้อ่านเขียนได้รวดเร็ว)
- **Version Control:** GitHub

---

## 📅 แผนการทำงาน (4 สัปดาห์)
- **สัปดาห์ที่ 1:** นั่งคุยกันว่าอยากได้ระบบแบบไหน ออกแบบขอบเขตของงาน
- **สัปดาห์ที่ 2:** เริ่มปั้นหน้าเว็บ (Frontend) ออกแบบหน้าตาให้สวยและใช้งานง่ายด้วย React
- **สัปดาห์ที่ 3:** ลุยระบบหลังบ้าน (Backend) ทำ API สำหรับจัดการสินค้าและเชื่อมต่อข้อมูลเข้ากับหน้าเว็บ
- **สัปดาห์ที่ 4:** เอามาเชื่อมต่อกันทั้งหมด ทดสอบระบบ หาบั๊ก แล้วก็จัดทำเอกสารส่งงานครับ

---

## 🧪 การทดสอบระบบ (Testing)
เพื่อให้แน่ใจว่าเว็บเราไม่พัง เราเลยเทสตามนี้ครับ:
- **Functional Testing:** ลองกดเล่นฟังก์ชันหลักๆ ว่าทำงานถูกมั้ย (สั่งซื้อได้จริงเปล่า, สต็อกลดมั้ย)
- **เครื่องมือที่ใช้:** Postman (เอาไว้ยิงเทส API หลังบ้าน), Browser DevTools (เอาไว้ดูความผิดปกติหน้าเว็บ)
