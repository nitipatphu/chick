## 🎯 วัตถุประสงค์

เพื่อพัฒนาระบบจำหน่ายอุปกรณ์กีฬาออนไลน์ ที่มีวัตถุประสงค์ดังนี้:

1. เพื่อพัฒนาระบบจำหน่ายอุปกรณ์กีฬาออนไลน์
2. เพื่อให้ลูกค้าสามารถค้นหาและสั่งสินค้าได้สะดวก
3. เพื่อช่วยผู้ดูแลลระบบจัดการสินค้าและคำสั่งชื้อได้สะดวก

---

## 📌 หลักการและเหตุผล

ปัจจุบันการชื้อขายสินค้าออนไลน์ได้รับความนิยมอย่างมากโดยเฉพาะอุปกรณ์กีฬามากมายหลายประเภทผมจึงอยากทำระบบจำหน่ายอุปกรณ์กีฬาเพื่ออำนวยความสะดวกให้ลูกค้าสามารถ
ตรวจสอบรายละเอียดและสั่งชื้อผ่านระบบรวมถึงให้ผู้ดูแลจัดการข้อมูลได้อย่างมีประสิทธิภาพ

## 👤 ขอบเขตระบบ (System Scope)

### Actors (ผู้ที่เกี่ยวข้อง)

| Actor | บทบาท | หน้าที่หลัก |
|-------|-------|-----------|
| **Customer** | ผู้ใช้งานทั่วไป | - ค้นหาและเรียกดูสินค้า<br>- เลือกซื้อสินค้า<br>- ตรวจสอบสถานะคำสั่งซื้อ<br>- จัดการโปรไฟล์ส่วนตัว |
| **Staff** | พนักงาน | - จัดการคำสั่งซื้อ<br>- ตรวจสอบและอัปเดตสต็อกสินค้า<br>- จัดการข้อมูลสินค้าบางส่วน |
| **Administrator** | ผู้ดูแลระบบ | - จัดการข้อมูลสินค้าทั้งหมด (CRUD)<br>- จัดการข้อมูลผู้ใช้งาน<br>- ควบคุมระบบและสิทธิ์การเข้าถึง<br>- ดูรายงานสรุปยอดขาย |

---

### ความสามารถหลักของระบบ

#### สำหรับ Customer
- ลงทะเบียนและเข้าสู่ระบบ
- ค้นหาสินค้าตามหมวดหมู่และคำค้น
- ดูรายละเอียดสินค้า
- เพิ่มสินค้าในตะกร้า
- สั่งซื้อสินค้า
- ตรวจสอบประวัติการสั่งซื้อ

#### สำหรับ Staff
- เข้าสู่ระบบด้วยสิทธิ์พนักงาน
- จัดการคำสั่งซื้อ (อัปเดตสถานะ)
- ตรวจสอบสินค้าคงคลัง
- จัดการข้อมูลสินค้า (เพิ่ม/แก้ไข)

#### สำหรับ Administrator
- เข้าสู่ระบบด้วยสิทธิ์ผู้ดูแล
- จัดการข้อมูลสินค้าทั้งหมด (เพิ่ม/แก้ไข/ลบ)
- จัดการข้อมูลผู้ใช้งาน
- ดูรายงานและสถิติ
- ตั้งค่าระบบ

## 📊 ผลลัพธ์ที่คาดว่าจะได้รับ

1. ได้ระบบจำหน่ายอุปกรณ์กีฬาสามารถใช้งานได้จริง
2. ผู้ใช้งาน (Customer) สามารถเลือกซื้อสินค้าผ่านระบบได้อย่างสะดวกและรวดเร็ว
3. พนักงาน (Staff) สามารถจัดการคำสั่งซื้อและสินค้าคงคลังได้อย่างมีประสิทธิภาพ
4. ผู้ดูแลระบบ (Administrator) สามารถบริหารจัดการระบบและข้อมูลทั้งหมดได้
5. ลดความผิดพลากในการจัดข้อมูลสินค้าและคำสั่งชื้อ

## 📅 แผนการดำเนินงาน (4 สัปดาห์)

| สัปดาห์ | กิจกรรม | รายละเอียด |
|---------|---------|-----------|
| **สัปดาห์ที่ 1** | วางแผนและวิเคราะห์ (Analysis & Design) | -ศึกษาตามความต้องการระบบออกแบบขอบเขต |
| **สัปดาห์ที่ 2** | ออกแบบและพัฒนาหน้าจอ (Frontend) | - ออกแบบ UX/UI<br> |
| **สัปดาห์ที่ 3** | พัฒนาระบบหลังบ้าน (Backend Development) | - พัฒนาระบบจัดการสินค้า<br>- เชื่อมต่อฐานข้อมูล Local Storage<br>- เชื่อมต่อ Frontend กับ Backend |
| **สัปดาห์ที่ 4** | ทดสอบและปรับปรุง (Testing & Deployment) | - ทดสอบระบบและแก้ไขข้อบกพร่อง<br>- ปรับปรุงประสิทธิภาพ<br>- จัดทำเอกสารและส่งมอบงาน |

## 🔄 แนวทางการพัฒนาตาม SDLC

| ขั้นตอน | รายละเอียด | กิจกรรมหลัก |
|---------|-----------|-----------|
| **1. Planning** | การวางแผนโครงการ | - วางแผนกำเหนดขอบเขต |
| **2. Analysis** | การวิเคราะห์ความต้องการ | - วิเคราะห์ตามความต้องการผู้ใช้ |
| **3. Design** | การออกแบบระบบ | - ออกแบบสถาปัตยกรรมระบบ<br>- ออกแบบฐานข้อมูล<br>- ออกแบบ UI/UX |
| **4. Development** | การพัฒนา | - พัฒนา Frontend ด้วย React<br>- พัฒนา Backend ด้วย Java<br>- เชื่อมต่อระบบและฐานข้อมูล |
| **5. Testing** | การทดสอบระบบ | - ทดสอบ User Acceptance Testing (UAT) |
| **6. Deployment** | การติดตั้งและใช้งาน | - ติดตั้งระบบบนเซิร์ฟเวอร์<br>- ตรวจสอบความพร้อมก่อนใช้งานจริง<br>- ส่งมอบระบบให้ผู้ใช้งาน |
| **7. Maintenance** | การบำรุงรักษา | - แก้ไขข้อผิดพลาดที่พบภายหลัง<br>- ปรับปรุงและพัฒนาระบบต่อเนื่อง<br>- ดูแลระบบให้ทำงานได้อย่างเสถียร |
